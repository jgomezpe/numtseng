Un identificador es una secuencia de símbolos que utilizamos como nombre para las variables, funciones, arreglos, clases y otras estructuras propias de los lenguajes de programación. Los identificadores en Python los escribimos como secuencias de caracteres alfanuméricos (letras del abecedario inglés o los dígitos) o el guión bajo (_), secuencias en las que el primer símbolo no puede ser un dígito y que deben tener al menos una letra. Los identificadores en Python deben tener una longitud mayor o igual a uno.

Adicionalmente, un identificador no debe ser una palabra reservada del lenguaje. En Python las palabras reservadas son las siguientes:
and as assert break class
continue def del elif else
except finally False for from
global if import in is
lambda nonlocal None not or
pass raise return True try
with while yield
Tabla. Palabras reservadas en Python.

Notemos que los identificadores para los tipos de datos en Python, por ejemplo el identificador para los números enteros (int), no son considerados palabras reservadas, esto quiere decir que podríamos usarlos como cualquier otro identificador. Sin embargo, en este curso, los trataremos y sugerimos que los consideremos como si fueran palabras reservadas.

Es importante además que sepamos que Python es un lenguaje de programación sensible a mayúsculas y minúsculas, esto quiere decir que en los identificadores no es lo mismo una letra mayúscula a su correspondiente en minúscula, por ejemplo los identificadores dia, Dia y DIA pueden referirse a tres entidades diferentes (variables, funciones, etc.) en un mismo programa ya que aunque son la misma palabra, difieren en el uso de letras mayúsculas y/o minúsculas.

Por cierto, existen lenguajes que no son sensibles a mayúsculas y minúsculas, por ejemplo BASIC, FORTRAN y HTML.
